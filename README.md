# 低位首板回踩缩量战法 · 选股 + 信号体检工具

一个**纯前端、离线、免费**的股票战法工具：用 Python 脚本把 A股 / 港股 / 美股 的日线行情下载成本地 JSON，再用浏览器打开 `index.html` 做**每日选股扫描**和**信号体检**。无需服务器，无需注册任何账号。

---

## 1. 这是什么

**战法一句话（A股 / 港股）**：在长期横盘的**低位**股票里，抓它的**第一次涨停（D0）**；涨停后观察 2~3 天，如果**没跌破涨停日收盘价**且**成交量缩量**，就在次日**开盘买入**；之后**涨 20% 止盈、跌 5% 止损**，最长持有 30 个交易日。

- 港股没有涨停板，用「**单日大涨阈值**」（默认 9%）替代涨停作为触发信号。
- A股按板块自动套用涨停幅度：主板 10% / 创业板·科创板 20% / 北交所 30%，名称含 ST 的用 4.8%。
- **美股走的是另一套独立策略**（平台 / 杯柄突破），不套用涨停逻辑，详见第 9 节。

工具做两件事：

1. **选股扫描** —— 在最新数据里找出**正处于「观察期」或已「可买入」**的股票，供你次日开盘操作。
2. **信号体检** —— 对全市场历史数据逐笔模拟这套战法，统计**它作为信号本身的质量**：胜率、平均收益、盈亏比、退出方式分布（止盈 / 止损 / 超时各占多少）；美股再叠加一层**参考价值分层**。它回答的是「这套信号靠不靠谱」，**不是**「我拿一笔钱跑下来赚了多少」——因此不再画资金曲线、不算累计收益和最大回撤。

> 提示：信号体检是简化撮合，不计手续费、滑点、涨跌停无法成交等真实约束，结果仅供研究参考，详见第 11 节。

> **当前工具只显示「港股 / 美股」两个市场，A股 已隐藏。** A股 的战法引擎、下载脚本（`fetch_a_baostock.py`，含北交所补充与退市股等处理）和数据都仍然保留，随时可用；只是页面顶部的 A股 切换按钮被隐藏、默认打开港股。如需重新显示 A股，在 `index.html` 顶部 `#marketSeg` 里加回 `<button data-m="A">A股</button>`、并把 `state` 的 `market:'HK'` 改回 `'A'` 即可。

## 3. 环境与依赖安装

### 3.1 前置

- 一台装有 **Python 3** 的电脑（macOS 自带 `python3`）。
- 任意现代浏览器（Chrome / Edge / Safari 等）打开 `index.html`。

### 3.2 安装下载脚本依赖

推荐方案（A股用 baostock，港股/美股用 yfinance）：

```bash
pip3 install baostock yfinance openpyxl --break-system-packages
```

- `baostock`：A股行情。
- `yfinance`：港股 / 美股行情。
- `openpyxl`：解析港交所 HKEX 官方证券列表的 `.xlsx` 文件（`fetch_hkus_yf.py` 取港股代码时用）。缺它时脚本会自动退回东财 / 内置蓝筹种子列表，但装上才能拿到最全的 ~2700+ 只港股。

> **为什么要加 `--break-system-packages`？**
> macOS 自带的系统 Python 受 PEP 668 保护，直接 `pip3 install` 会被拒绝（提示 externally-managed-environment）。加上 `--break-system-packages` 即允许装到用户环境。如果你更习惯用虚拟环境，也可以 `python3 -m venv venv && source venv/bin/activate` 后再正常 `pip install`，效果相同。

如果只想用**备用脚本** `fetch_data.py`（东方财富），**无需安装任何依赖**，它只用 Python 标准库。

## 5. 下载脚本命令与常用参数

> 三个脚本都把结果写到**脚本所在目录**，文件名固定为 `data_A.json` / `data_HK.json` / `data_US.json`，会覆盖旧文件。

### 5.1 `fetch_a_baostock.py`（A股，推荐）

价格为**前复权**（与战法低位/横盘口径一致）。baostock 是开源、免注册的批量历史行情库，全市场约 5000+ 只个股，**首次全量**下载耗时视网络而定（约 10~30+ 分钟，因为免费接口是逐只股票请求，没有"一次性拿全市场"的批量接口）。

| 参数 | 默认 | 说明 |
|---|---|---|
| `--limit N` | 无 | 只下前 N 只（调试用） |
| `--days N` | 450 | **首次全量**时每只回溯的自然日数（450 ≈ 300 个交易日） |
| `--incremental` | 关 | **增量模式**：已收录的股票只请求"本地最新日期+1 到今天"这一小段并追加合并；只有本地没有的新股票才全量拉取。每天收盘后用这个，比首次全量快很多（详见第 6 节） |

```bash
python3 fetch_a_baostock.py                 # 首次：下全部 A股（含北交所）
python3 fetch_a_baostock.py --limit 30      # 调试：只下前 30 只
python3 fetch_a_baostock.py --days 400      # 自定义首次回溯天数
python3 fetch_a_baostock.py --incremental   # 日常：增量更新（收盘后每天跑这个）
```

> 单只股票本地最多保留最近 500 个交易日（约 2 年），增量模式下超出部分会自动裁剪，避免 `data_A.json` 无限增长。

### 5.2 `fetch_hkus_yf.py`（港股 / 美股，推荐）

数据源 yfinance（雅虎财经，开源免注册）。美股代码来自 NASDAQ Trader 官方表（`nasdaqlisted` + `otherlisted`，自动剔除 ETF 和测试票，只保留个股）；港股代码来自港交所 HKEX 官方证券列表（`.xlsx`，只取主板 + GEM 的股本证券，剔除权证 / 牛熊证 / 债券 / ETF），需要 `openpyxl`，取代码时按 **① HKEX 官方 → ② 东财 clist → ③ 内置蓝筹种子** 三层兜底。此外，下载美股时会额外抓取大盘指数 **SPY** 的日线，写入 `data_US.json` 的 `index` 字段，供美股策略的「大盘没有走弱」过滤使用。

| 参数 | 默认 | 说明 |
|---|---|---|
| `--markets` | `HK,US` | 要下载的市场，逗号分隔 |
| `--period` | `2y` | **首次全量**回溯时长，如 `1y` / `2y` / `3y`（低位窗口需 ≥ 250 个交易日，建议**不要低于 2y**） |
| `--limit N` | 无 | 每个市场只下前 N 只（调试用） |
| `--batch N` | 40 | yfinance 批量下载的每批数量 |
| `--incremental` | 关 | **增量模式**，同 `fetch_a_baostock.py`：已收录的股票按本地最新日期分组，只补最新几天；新股全量拉取 |

```bash
python3 fetch_hkus_yf.py                      # 首次：港股 + 美股全量
python3 fetch_hkus_yf.py --markets HK         # 只下港股
python3 fetch_hkus_yf.py --markets US --limit 50   # 美股前 50 只（调试）
python3 fetch_hkus_yf.py --period 3y          # 回溯 3 年
python3 fetch_hkus_yf.py --incremental        # 日常：增量更新
```

> 同样单只股票本地最多保留最近 500 个交易日，增量模式下自动裁剪。

### 5.3 `fetch_data.py`（东方财富，免安装备用）

只用标准库，无需 pip。但东方财富 K 线接口在批量拉全市场时容易**限频 / 断连**，所以仅作备用方案。

| 参数 | 默认 | 说明 |
|---|---|---|
| `--markets` | `A,HK,US` | 要下载的市场，逗号分隔 |
| `--limit N` | 无 | 每个市场最多下 N 只（调试用） |
| `--workers N` | 6 | 并发线程数（过高更易被限频断连） |
| `--us-active-only` | 关 | 美股只下成交额最活跃的前 N 只，避免上万只下太久 |
| `--us-active-limit N` | 1500 | 配合上面，控制美股活跃股数量 |

```bash
python3 fetch_data.py                          # 下全部三个市场
python3 fetch_data.py --markets A              # 只下 A股
python3 fetch_data.py --us-active-only --us-active-limit 1500   # 美股只下最活跃 1500 只
```

## 6. 每天怎么用

**第一次**必须全量下载（本地还没有历史数据可参照）：
```bash
cd /Users/lihaojun/Desktop/zhanfa
python3 fetch_a_baostock.py      # A股 首次全量（较慢，几千只 x 完整历史）
python3 fetch_hkus_yf.py         # 港股 + 美股 首次全量
# 或用一键脚本：python3 update_all.py
```

**之后每天**收盘后（A股 15:00 之后），改用 `--incremental` 增量更新——已收录的股票只补"本地最新日期到今天"这一两天数据并追加，不会重新下载几年历史，明显更快：
```bash
cd /Users/lihaojun/Desktop/zhanfa
python3 fetch_a_baostock.py --incremental      # A股
python3 fetch_hkus_yf.py --incremental         # 港股 + 美股
# 或一键：python3 update_all.py --incremental
```

> 为什么每天还是要挨个请求：baostock / yfinance 都是免费开源方案，但接口设计是"一只股票一次请求"，没有免费的"一次性拿全市场当天行情"批量接口（这是免费数据源的共性限制）。增量模式省下的是**重复下载历史**的时间和流量，而不是请求次数——不过对"每天想拿当天最新全市场数据做选股 + 信号体检"这个需求来说已经足够，且比每天全量重下快很多。

日常操作流程：
1. **收盘后跑增量更新**（如上）。
2. **双击 `启动工具.command`**（或若浏览器已开着，直接刷新页面）——通过本地服务器打开时页面会**自动读取**当天最新的 `data_*.json`，无需手动选文件。（只有直接双击 `index.html` 以 `file://` 方式打开时才读不到盘、只显示内置样本，需要用顶部「加载数据文件」按钮手动选入。）
3. 在「**① 选股扫描**」看结果：
   - **可买入**：观察期已满足全部条件，**次日开盘**按战法买入。
   - **观察中**：D0 后观察期还没走完，先记下来，明天刷新数据后再看它转成「可买入」还是被淘汰。
4. **次日开盘按战法操作**：对「可买入」的票开盘买入，买入后设好 **+20% 止盈 / −5% 止损**，最长持有 30 个交易日到期则离场。
5. 切换顶部「A股 / 港股 / 美股」标签可分别加载对应市场的数据查看。

## 7. 战法参数面板说明

在 `index.html` 顶部点开「**战法参数（点击展开调整）**」即可逐项修改，改完点「**应用参数并重算**」生效，「**恢复默认**」可还原。以下为 `DEFAULT_PARAMS` 全部参数与默认值：

### D0 大涨阈值（触发「涨停 / 单日大涨」的当日涨幅，单位 %）

| 参数（界面名） | 默认值 | 含义 |
|---|---|---|
| A股主板 (`thMainA`) | 9.8 | 主板触发阈值（≈涨停 10%） |
| A股创业板/科创 (`thGemA`) | 19.6 | 创业板·科创板触发阈值（≈涨停 20%） |
| A股北交所 (`thBseA`) | 29.5 | 北交所触发阈值（≈涨停 30%） |
| A股 ST (`thStA`) | 4.8 | 名称含 ST/*ST 的触发阈值（≈涨停 5%） |
| 港股 (`thHK`) | 9 | 港股单日大涨阈值（无涨停，用此替代） |
| 美股 (`thUS`) | 9 | 美股单日大涨阈值（无涨停，用此替代） |

> 当日涨幅 = (今收 − 昨收) / 昨收 × 100，≥ 对应阈值即为触发日 D0。A股的板块按代码自动识别，ST 按名称自动识别。

### 形态条件（判断「首板 + 低位 + 横盘」）

| 参数（界面名） | 默认值 | 含义 |
|---|---|---|
| 首板回看天数 (`firstBoardLookback`) | 60 | D0 往前 N 天内不能再有触发日，确保是「第一次」涨停 |
| 低位窗口 (`lowWindow`) | 250 | 计算高低区间所用的回看交易日数 |
| 低位分位 (`lowPct`) | 30 | D0 收盘价须 ≤ 区间最低 + (最高−最低)×30%，即处于近期低 30% 位置 |
| 横盘窗口 (`flatWindow`) | 30 | 计算 D0 之前横盘振幅所用的回看交易日数 |
| 横盘振幅阈值 (`flatAmp`) | 25 | D0 之前窗口内 (最高−最低)/最低×100 须 < 25%，即前期足够平稳 |

### 观察 + 买入

| 参数（界面名） | 默认值 | 含义 |
|---|---|---|
| 观察天数 (`obsDays`) | 3 | D0 后观察的天数，期间收盘价（或最低价）不得跌破 D0 收盘价 |
| 缩量比 (`shrinkRatio`) | 0.7 | 观察末日成交量 / D0 成交量 须 < 0.7，即明显缩量 |
| 不跌破基准 (`notBreakUseLow`) | 收盘价 | 判断「是否跌破 D0 收盘价」时用**收盘价**还是**最低价**作比较；选最低价更严格 |

> 观察期全部满足后，在观察期之后的**次一交易日以开盘价买入**。

### 交易

| 参数（界面名） | 默认值 | 含义 |
|---|---|---|
| 止盈 (`takeProfit`) | 20 | 相对买入价上涨 20% 止盈 |
| 止损 (`stopLoss`) | 5 | 相对买入价下跌 5% 止损 |
| 最长持有 (`maxHold`) | 30 | 最多持有 30 个交易日，到期未触发止盈止损则按收盘价离场（超时） |

> 同一天若同时触及止盈与止损价，按**止损**处理（保守）。

### 扫描

| 参数（界面名） | 默认值 | 含义 |
|---|---|---|
| 扫描最近天数 (`scanRecent`) | 5 | 选股扫描只关注最近这么多天内出现的 D0 信号，确保是「当下可操作」的票 |

### 仅美股：平台 / 杯柄突破形态

> 以上参数用于 A股 / 港股的涨停战法；美股走的是**独立引擎**（第 9 节），下面这些参数**只对美股生效**。

| 参数（界面名） | 默认值 | 含义 |
|---|---|---|
| 整理窗口 (`usBaseWindow`) | 50 | 突破日往前 N 天作为「平台 / 杯柄整理区间」，其最高价即突破位 |
| 整理振幅上限 (`usBaseMaxAmp`) | 35 | 整理区间 (最高−最低)/最低×100 须 < 35%，即前期足够横盘 |
| 突破放量倍数 (`usBreakoutVolMult`) | 1.5 | 突破日成交量须 ≥ 整理区间日均量 × 1.5（放量） |
| 收盘位置下限 (`usBreakoutClosePos`) | 75 | 突破日收盘须落在当日 (收−低)/(高−低) ≥ 75% 处，即收在当日高位 |
| 突破幅度下限 (`usBreakoutAbovePct`) | 0 | 突破日收盘须高于整理区间高点这么多 %（0 = 只要收盘价高于高点即可） |

### 仅美股：回踩确认 + 大盘过滤

| 参数（界面名） | 默认值 | 含义 |
|---|---|---|
| 回踩观察天数 (`usPullbackDays`) | 5 | 突破后观察 N 天，期间不得跌破突破位 |
| 回踩缩量倍数 (`usPullbackVolMult`) | 0.7 | 观察末日量 / 突破日量 须 < 0.7（回踩缩量） |
| 允许跌破突破位 (`usPullbackHoldPct`) | 0 | 允许收盘跌破突破位的容忍度 %（0 = 完全不允许跌破） |
| 回踩不破基准 (`usPullbackUseLow`) | 收盘价 | 判断「是否跌破突破位」用收盘价还是最低价（选最低价更严格） |
| 大盘过滤 (`usMarketFilter`) | 启用 | 突破当日若大盘 SPY 走弱则不采纳该信号（见下） |
| 大盘均线周期 (`usMarketSma`) | 50 | 大盘「没有走弱」判定：SPY 突破日收盘 ≥ 其 N 日均线 |

### 仅美股：交易 + 参考价值分层

| 参数（界面名） | 默认值 | 含义 |

### ① 选股扫描

对当前市场全部股票扫描，列出**正处于「观察期 / 可买入」**的股票表格，每行包含：

### ② 信号体检

对当前市场全部股票、在选定日期区间内逐笔模拟这套战法，评估**信号本身的质量**（不是模拟一笔资金滚动，所以没有资金曲线 / 累计收益 / 最大回撤）。可选「区间」起止日期，或点「全部历史」。输出：

- **统计卡片**：总信号次数、**胜率**、**平均收益率**、**盈亏比**（均盈 / 均亏）；下方附盈 / 亏笔数与均盈 / 均亏。
- **退出方式分布**：所有信号中，最终以**止盈 / 止损 / 超时**退出的各占多少——直观看出这套信号更多是「赚够止盈」还是「拖到超时 / 打到止损」。
- **美股参考价值分层**（仅美股）：把信号按「较高 / 中等偏高 / 中等 / 较低」分组分别看质量，帮你判断哪一层的突破信号更值得跟（分层规则见第 9 节）。

## 9. 美股：独立的「平台 / 杯柄突破」策略

美股**不套用**A股 / 港股的涨停战法，而是走一套单独的引擎（在 `index.html` 里与涨停逻辑完全分开）。一句话：**长期横盘 / 杯柄整理 → 放量突破平台 → 收盘收在当天高位 → 之后几天缩量回踩、不跌破突破位 → 期间大盘（SPY）没有走弱 → 次日开盘买入**。

规则要点：

- **突破位** = 突破日往前 `usBaseWindow`(50) 天整理区间的**最高价**。整理区间振幅要够小（< `usBaseMaxAmp` 35%），确认是横盘 / 杯柄而非已在拉升。
- **放量突破**：突破日量 ≥ 区间日均量 × `usBreakoutVolMult`(1.5)，且收盘收在当日高位（收盘位置 ≥ `usBreakoutClosePos` 75%）、收盘高于突破位。
- **缩量回踩**：突破后 `usPullbackDays`(5) 天内不跌破突破位，且末日缩量（末日量 / 突破日量 < `usPullbackVolMult` 0.7）。
- **大盘过滤（SPY）**：突破当日若大盘 SPY 收盘 < 其 `usMarketSma`(50) 日均线，视为大盘走弱，**不采纳**该信号（`usMarketFilter` 可关）。SPY 日线随美股数据一起下载，存于 `data_US.json` 的 `index` 字段；若数据里没有 index，则此条不拦截。
- **止损锚在突破位下方**：止损价 = 突破位 ×(1 − `usStopBelowBreakoutPct`/100)，即**突破位下方 3%**，而不是机械的「买入价 −5%」——突破位是这套形态的技术失效位，跌破它才算走坏。止盈 `usTakeProfit`(20)%、最长持有 `usMaxHold`(60) 日。

**参考价值分层**（对突破信号打「值不值得跟」的标签）：

- 整理区间**日均成交额** ≥ `usLargeCapDollarVolM`(30) 百万美元 → **较高**（代理大盘股 / 龙头，流动性与确定性更好）。
- 日均成交额 ≥ `usMidCapDollarVolM`(3) 百万美元 → **中等**；若该股同期涨幅还**跑赢大盘**（相对强弱 RS > 0）→ 升为**中等偏高**。
- 两档都够不上 → **较低**。

> **一处诚实说明（板块强弱用相对大盘强弱代理）**：真正的「所在板块 / 行业是否强势」需要行业分类和板块 ETF 数据，本工具的免费数据源里没有。因此这里用**个股同期涨幅相对大盘（SPY）涨幅**的相对强弱（RS）来**代理**「强势与否」——这是量化里常见的替代做法，但**不等价于**真实的行业 / 板块强弱，仅供参考。
